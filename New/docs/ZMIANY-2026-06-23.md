## 2. Backend — Appwrite Functions

Wszystkie trasy wywołuje się przez **jedną funkcję Appwrite** (`NEXT_PUBLIC_APPWRITE_FUNCTION_ID`), z nagłówkiem sesji użytkownika (cookie JWT Appwrite).

### Wspólny format wywołania (frontend)

```typescript
import { apiRequest } from '@/lib/api-client';

const result = await apiRequest<ResponseType>(
  '/auth/me',      // ścieżka (_route)
  'GET',           // metoda HTTP
  { /* body */ },  // opcjonalne — pomijane przy GET
  true             // requireAuth (domyślnie true)
);
```

Odpowiedź:

```json
{
  "success": true,
  "...": "dane specyficzne dla endpointu"
}
```

Błąd:

```json
{
  "success": false,
  "code": "ERROR_CODE",
  "error": "Opis błędu"
}
```

### `GET /health`

| | |
|---|---|
| **Auth** | Nie |
| **Opis** | Sprawdzenie, czy funkcja działa i API key jest skonfigurowany |

**Odpowiedź:** `{ success: true, version, apiKeyConfigured, path }`

---

### `GET /auth/me`

| | |
|---|---|
| **Auth** | Tak (sesja Appwrite) |
| **Opis** | Pełny profil Quark + integracja Steam |

**Odpowiedź:**

```json
{
  "success": true,
  "profile": { /* UserProfile */ },
  "steamIntegration": { /* SteamIntegration | null */ }
}
```

**Naprawa:** `parseBody` nie jest wywoływany na GET (wcześniej powodowało `FUNCTION_ERROR`).

---

### `PATCH /auth/profile`

| | |
|---|---|
| **Auth** | Tak |
| **Rate limit** | `profile/patch` |

**Body (wszystkie pola opcjonalne):**

| Pole | Typ | Opis |
|------|-----|------|
| `displayName` | `string` | max 32 znaki |
| `bio` | `string` | max 190 znaków, HTML stripowany |
| `cardTheme` | `string` \| `object` | JSON motywu karty |
| `customStatus` | `string` | max 128 znaków |
| `presence` | `'online' \| 'idle' \| 'dnd' \| 'offline'` | status obecności |
| `preferences` | `string` | JSON preferencji użytkownika |

**Uwaga:** `avatarFileId` **nie jest już akceptowane** w PATCH — avatar zmienia się wyłącznie przez `POST /auth/avatar`.

**Przykład — zapisanie flagi pominięcia modala Steam:**

```typescript
import { mergeProfilePreferences } from '@/lib/profile-preferences';

await updateProfile({
  preferences: mergeProfilePreferences(profile.preferences, {
    steamPromptSkipped: true,
  }),
});
```

**Odpowiedź:** `{ success: true, profile: UserProfile }`

---

### `POST /auth/avatar` *(NOWY)*

| | |
|---|---|
| **Auth** | Tak (tylko własny profil) |
| **Rate limit** | `avatar/upload` |
| **Opis** | Upload avatara przez serwer (API key) do bucketa `user_media` |

**Body:**

| Pole | Typ | Wymagane | Opis |
|------|-----|----------|------|
| `data` | `string` | tak | Obraz zakodowany **base64** (bez prefixu `data:...`) |
| `mimeType` | `string` | tak | `image/png`, `image/jpeg`, `image/jpg`, `image/webp`, `image/gif` |

**Limity:** max **5 MB** po dekodowaniu base64.

**Flow serwera:**
1. Walidacja typu i rozmiaru
2. Usunięcie starego pliku avatara (jeśli istnieje)
3. `storage.createFile` z uprawnieniami pliku: `read:any`, `update/delete:user:{userId}`
4. Aktualizacja `avatarFileId` w `user_profiles`

**Odpowiedź:**

```json
{
  "success": true,
  "fileId": "abc123...",
  "avatarUrl": "https://.../storage/buckets/user_media/files/{fileId}/view?project=...",
  "profile": { /* zaktualizowany UserProfile */ }
}
```

**Kody błędów:** `INVALID_AVATAR`, `AVATAR_TOO_LARGE`, `PROFILE_NOT_FOUND`, `RATE_LIMITED`, `UNAUTHORIZED`

---

### `POST /auth/steam/link`

Bez zmian w kontrakcie — używany po OpenID lub ręcznym podaniu Steam ID / vanity URL.

**Body:**

| Pole | Typ | Opis |
|------|-----|------|
| `steamId` | `string?` | 17-cyfrowe Steam ID |
| `vanityUrl` | `string?` | np. `gaben` z `steamcommunity.com/id/gaben` |

## 3. Avatary — upload i odczyt

### Architektura

```
┌─────────────┐     POST /auth/avatar      ┌──────────────────┐
│   Klient    │ ─────────────────────────► │ Appwrite Function│
│ (Next.js)   │     base64 + mimeType      │  (API key)       │
└─────────────┘                            └────────┬─────────┘
       │                                            │
       │  getAvatarUrl(fileId)                      │ createFile
       │  bezpośredni <img src>                     ▼
       └──────────────────────────────────► ┌──────────────────┐
                                            │ Bucket user_media │
                                            │ (brak create:user)│
                                            └──────────────────┘
```

- **Upload / zmiana:** wyłącznie przez API (`POST /auth/avatar`)
- **Odczyt:** bezpośredni URL Appwrite — **nie** przez API (znajomi, karty profilu, menu)

### `lib/avatar-service.ts`

#### `getAvatarUrl(fileId)`

```typescript
import { getAvatarUrl } from '@/lib/avatar-service';

const url = getAvatarUrl(profile.avatarFileId);
// => string | undefined
```

Zwraca publiczny URL widoku pliku. Wymaga `Permission.read(Role.any())` na pliku (ustawiane przy uploadzie).

#### `uploadAvatar(file)`

```typescript
import { uploadAvatar } from '@/lib/avatar-service';

const result = await uploadAvatar(fileFromInput);
// result: { success, fileId?, avatarUrl?, profile?, error? }
```

Wewnętrznie wywołuje `POST /auth/avatar`. **Nie** używa SDK Storage po stronie klienta.

#### `fileToPreviewUrl(file)`

```typescript
import { fileToPreviewUrl } from '@/lib/avatar-service';

const dataUrl = await fileToPreviewUrl(file);
// data:image/png;base64,... — podgląd przed uploadem (zgodne z CSP)
```

### `useAuth().applyProfile(profile)`

Po udanym uploadzie zaktualizuj stan kontekstu:

```typescript
const { applyProfile } = useAuth();
applyProfile(upload.profile);
```

Aktualizuje `profile` w React + cache offline (`quark_profile_cache`).

## 5. Integracja Steam

### `SteamIntegrationPanel`

**Plik:** `web/act-l/components/steam-integration-panel.tsx`

#### Główny flow — logowanie OpenID (Electron)

```typescript
import { loginWithSteam } from '@/lib/steam-openid';

const login = await loginWithSteam();
// { success, steamId?, error? }

if (login.success && login.steamId) {
  await linkSteam(login.steamId);
}
```

Przycisk UI: **„Zaloguj przez Steam”** — otwiera przeglądarkę Steam, callback na `http://127.0.0.1:38492/steam/callback`.

#### Flow zapasowy — ręczne połączenie

Rozwijane pod „Połączenie ręczne (zaawansowane)”:
- Steam ID (17 cyfr) lub URL profilu
- Wywołuje `linkSteam(steamId?, vanityUrl?)`

#### Po połączeniu

- `loadSteamData(steamId)` — znajomi, statystyki przez `callSteamApi`
- Sync do `settings-context` (`steamUser`, `steamFriends`)

### `SteamSync`

**Plik:** `web/act-l/components/steam-sync.tsx`

Synchronizuje `useAuth().steamIntegration` → `useSettings()` (steam user ID, avatar URL).

### Tłumaczenia Steam

Namespace: `steam` w `messages/pl.json` i `messages/en.json`  
Klucze: `loginButton`, `loginHint`, `manualLink`, `connected`, itd.

## 9. Electron — auto-aktualizacje

### Flow

```
did-finish-load → checkForUpdates()
       ↓
update-available → IPC update-available-to-ui → UpdateBanner
       ↓
Użytkownik klika „Aktualizuj” → startInstallation() → downloadUpdate()
       ↓
update-download-progress → pasek postępu w UI
       ↓
update-downloaded → quitAndInstall()
```

### Konfiguracja (`Windows app/main.js`)

| Ustawienie | Wartość |
|------------|---------|
| `autoDownload` | `false` (użytkownik decyduje) |
| `allowPrerelease` | `true` |
| `forceDevUpdateConfig` | `true` w dev (czyta `dev-app-update.yml`) |

**Wersja w `package.json`:** bez prefiksu `v` (np. `0.0.4-beta03c`) — wymagane przez semver `electron-updater`.

### `dev-app-update.yml`

```yaml
owner: c14b7
repo: Quark-Launcher

```

## 10. Electron — Steam OpenID

### IPC: `steam-openid-login`

**Plik:** `Windows app/main.js`

1. Uruchamia lokalny serwer HTTP na `127.0.0.1:38492`
2. Otwiera `https://steamcommunity.com/openid/login?...` w przeglądarce
3. Po callbacku parsuje Steam ID z `openid.claimed_id`
4. Zwraca `{ success: true, steamId: '76561198...' }`

Timeout: 5 minut.

### Klient

```typescript
// lib/steam-openid.ts
const result = await loginWithSteam();

// lub bezpośrednio:
const result = await window.electronAPI.steamOpenIdLogin();
```

**Poza Electronem:** zwraca błąd — użyj ręcznego połączenia w panelu Steam.

## 14. Wdrożenie po zmianach

### 1. Appwrite Function (wymagane)

```bash
cd functions
npm run build:compile
# Wgraj functions/dist/ do Appwrite Console lub: appwrite deploy function
```

Nowe/zmienione endpointy: `POST /auth/avatar`, poprawki GET, `PATCH` bez `avatarFileId`.

### 2. Bucket Storage (wymagane raz)

Uruchom `setup-database.ts` lub ręcznie usuń `create` dla `users` na `user_media`.

### 3. Frontend

```bash
cd web/act-l
npm run dev        # dev na :30211
npm run build      # export do out/ (Electron)
```

### 4. Electron

```bash
cd "Windows app"
npm run dev        # Next + Electron
npm run release    # build + publish GitHub (auto-update)
```

### Deploy funkcji — naprawa `InputFile`

**Problem:** `import { InputFile } from 'node-appwrite/file'` nie kompilował się na Appwrite Open Runtimes (`moduleResolution`).

**Rozwiązanie:** wrapper CJS w `functions/lib/input-file.ts` używający **oficjalnego exportu** pakietu:

```typescript
const inputFileModule = require('node-appwrite/file');
export const InputFile = inputFileModule.InputFile;
```

> Nie używaj `node-appwrite/dist/inputFile.js` — ten subpath nie jest w `exports` i runtime Appwrite go odrzuca.

W `auth-api.ts`: `import { InputFile } from './lib/input-file'`.

Build:

```bash
cd functions && npm run build:compile
```
